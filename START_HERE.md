# 🚀 一键启动指南（Mock 后端版）

## 方式一：使用 Mock 后端（推荐用于开发）

### 快速启动

```bash
# 1. 复制环境变量（首次运行需要）
cp .env.example .env.local

# 2. 安装依赖（如果还没安装）
pnpm install

# 3. 启动开发服务器（包含 Mock 后端）
pnpm dev
```

### 访问应用

- **前端**: http://localhost:3000
- **注册页面**: http://localhost:3000/register
- **登录页面**: http://localhost:3000/login

### 查看验证码

注册时，验证码会直接打印在**终端（命令行）**中：

```
📧 【注册验证码】
📮 邮箱：test@example.com
🔢 验证码：123456
⏰ 有效期：5 分钟
```

**无需真实邮箱！** 直接从终端获取验证码即可。



## 📋 完整测试流程（Mock 后端）


## 🔍 常见问题

### Q: 找不到验证码？

**A**: 验证码打印在你运行 `pnpm dev` 的终端窗口中。仔细查看命令行输出，找到类似这样的内容：

```
📧 【注册验证码】
📮 邮箱：test@example.com
🔢 验证码：123456
```

### Q: 提示"验证码错误"？

**A**: 确保输入的是终端中显示的**最新**验证码。每次点击"获取验证码"都会生成新的验证码。

### Q: 提示"验证码已过期"？

**A**: 验证码有效期为 5 分钟。过期后请重新点击"获取验证码"获取新的验证码。

### Q: 提示"该邮箱已注册"？

**A**: 每个邮箱只能注册一次。如需测试新用户，请使用不同的邮箱地址。

### Q: 如何清除所有数据重新开始？

**A**: 
```bash
# 停止开发服务器（Ctrl+C）
# 删除存储目录
rm -rf .nitro/dev/storage
# 重新启动
pnpm dev
```

## ⚠️ 重要提示

### Mock 后端限制

- ✅ 适合开发和测试
- ✅ 无需配置邮件服务
- ✅ 快速搭建和调试
